Nginx + PHP + SSL: веб-сервер и сертификаты

«Сайт должен быть открыт для гостя, но не для воровства. Как балконная дверь: свежий воздух — да, посторонние — нет. — Воланд»

Введение

Настоящее руководство описывает установку и настройку веб-сервера Nginx с интерпретатором PHP и сертификатами Let's Encrypt для обеспечения безопасного HTTPS-соединения.

Мы начинаем с сервера, на котором уже выполнены предыдущие шаги: настроена базовая безопасность, UFW, уведомления по почте. Теперь сервер должен начать обслуживать сайт.

Цель данного руководства — настроить Nginx так, чтобы сайт на Grav CMS работал по HTTPS (порт 8443), а HTTP-запросы (порт 80 → sslh → 8080) перенаправлялись на защищённое соединение.

Входные данные: у вас есть домен (например, psj-notes.ru), сервер доступен по SSH, домен делегирован на ваш хостинг.

Раздел 1. Что мы настраиваем

1.1. Архитектура

Пользователь → http://psj-notes.ru:80
                           ↓ Timeweb балансировщик
                           ↓ сервер:80
                           ↓ sslh (определяет HTTP)
                           ↓ 127.0.0.1:8080 → Nginx HTTP
                                           ↓ редирект 301
                             https://psj-notes.ru:8443 → Nginx HTTPS
                                                     ↓ PHP-FPM
                                                     ↓ Grav CMS
Компонент Роль
Nginx HTTP (8080) Принимает трафик через sslh, отдаёт 301 редирект на HTTPS
Nginx HTTPS (8443) Обслуживает Grav CMS, терминация SSL
Nginx x-ui (4443) Панель управления Xray (Basic Auth)
PHP-FPM Интерпретация PHP-скриптов
Let's Encrypt (certbot) Автоматическое получение/обновление SSL-сертификатов

Раздел 2. Установка Nginx и PHP

2.1. Установка Nginx

sudo apt install -y nginx

После установки Nginx запустится автоматически и будет слушать порт 80 на всех интерфейсах. Нам это не нужно — сайт будет работать через sslh на порту 8080, поэтому сразу остановим дефолтный сервер:

sudo systemctl stop nginx
sudo systemctl disable nginx  # Отключаем автозапуск — будем управлять сами

2.2. Установка PHP

Для Grav CMS нужен PHP 8.1 или новее. На момент написания актуальна PHP 8.4:

sudo apt install -y php-fpm php-mbstring php-curl php-xml php-zip php-gd php-mysql php-bcmath php-intl php-imagick php-apcu

Проверьте версию:

php -v

PHP-FPM запустится как systemd-сервис:

systemctl status php8.4-fpm

Запомните имя сокета — оно понадобится в конфиге Nginx:

ls /var/run/php/php*-fpm.sock

Раздел 3. Структура конфигурации Nginx

Nginx хранит конфигурацию в /etc/nginx/:

Файл/директория Назначение
nginx.conf Основной конфигурационный файл
sites-available/ Все виртуальные хосты (серверные блоки)
sites-enabled/ Активные виртуальные хосты (симлинки в sites-available/)
conf.d/ Дополнительные конфигурационные фрагменты

Правильный workflow: создать файл в sites-available/, затем создать симлинк в sites-enabled/.

Раздел 4. Настройка Nginx HTTP (порт 8080)

Сервер HTTP будет принимать трафик только на локальном интерфейсе, куда его направляет sslh. Его задача — отдать 301 редирект на HTTPS.

Создайте /etc/nginx/sites-available/grav-http:

server {
    listen 127.0.0.1:8080 default_server;
    server_name _;
    root /var/www/psj-notes.ru;

    location /.well-known/acme-challenge/ {
        allow all;
        default_type "text/plain";
    }
}

server {
    listen 127.0.0.1:8080;
    server_name psj-notes.ru www.psj-notes.ru consol.psj-notes.ru;
    root /var/www/psj-notes.ru;

    location /.well-known/acme-challenge/ {
        allow all;
        default_type "text/plain";
    }

    location / {
        return 301 https://$host$request_uri;
    }
}

Первый блок (default_server) обрабатывает запросы с неизвестным Host-заголовком (например, сканеры, боты) — он просто отдаёт корневую директорию. Второй блок — основной домен.

Раздел 5. Получение SSL-сертификата Let's Encrypt

5.1. Установка certbot

sudo apt install -y certbot

5.2. Получение сертификата

Поскольку порт 80 занят sslh, используем webroot-метод. Certbot создаст временный файл в .well-known/acme-challenge/, а Let's Encrypt проверит его доступность:

sudo certbot certonly --webroot -w /var/www/psj-notes.ru -d psj-notes.ru -d www.psj-notes.ru

Если всё прошло успешно, сертификаты будут в /etc/letsencrypt/live/psj-notes.ru/:

Файл Назначение
fullchain.pem Цепочка сертификатов (сертификат + промежуточные)
privkey.pem Приватный ключ
chain.pem Промежуточные сертификаты
cert.pem Только сертификат (без цепочки)

Раздел 6. Настройка Nginx HTTPS (порт 8443)

Создайте /etc/nginx/sites-available/grav-https:

server {
    listen 8443 ssl;
    server_name psj-notes.ru www.psj-notes.ru;
    root /var/www/psj-notes.ru;
    index index.html index.php;

    ssl_certificate /etc/letsencrypt/live/psj-notes.ru/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/psj-notes.ru/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    client_max_body_size 100M;

    # Grav security — закрыть служебные директории
    location ~* ^/(\.git|cache|bin|logs|backup|tests)/(.*)$ { deny all; }
    location ~* ^/(system|vendor|composer\.(json|lock))$ { deny all; }

    location ~ \.php$ {
        include fastcgi_params;
        fastcgi_pass unix:/var/run/php/php8.4-fpm.sock;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        fastcgi_param PATH_INFO $fastcgi_path_info;
    }

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }
}

6.1. Ключевые директивы

Директива Значение Почему
listen 8443 ssl Порт 8443 с SSL Основной порт 443 занят xray
ssl_protocols TLSv1.2 TLSv1.3 Только безопасные протоколы SSLv3, TLSv1.0/1.1 устарели
client_max_body_size 100M Максимальный размер загрузки Для изображений в Grav
fastcgi_pass unix:...sock Передача PHP через Unix-сокет Быстрее, чем TCP-локальный

Раздел 7. Активация и проверка

7.1. Активируйте сайты

sudo ln -s /etc/nginx/sites-available/grav-http /etc/nginx/sites-enabled/
sudo ln -s /etc/nginx/sites-available/grav-https /etc/nginx/sites-enabled/

7.2. Проверьте конфигурацию

sudo nginx -t

Если ошибок нет — перезагрузите Nginx:

sudo systemctl reload nginx

7.3. Проверьте работу сайта

# Локально — HTTP
curl -H "Host: psj-notes.ru" http://127.0.0.1:8080
# Должен вернуть 301 редирект

# Локально — HTTPS
curl -k https://127.0.0.1:8443
# Должен вернуть HTML-код Grav

Раздел 8. Настройка балансировщика Timeweb

8.1. HTTP

В панели Timeweb добавьте домен psj-notes.ru, укажите IP сервера и порт 80. Балансировщик направит запросы на ваш сервер:80, где sslh определит протокол и передаст Nginx на 8080.

8.2. HTTPS

Для HTTPS укажите порт 8443. Балансировщик будет принимать HTTPS-соединения и проксировать их на ваш сервер:8443.

Раздел 9. Автообновление сертификатов

Let's Encrypt сертификаты живут 90 дней. Certbot добавляет задачу в cron для автоматического обновления:

sudo certbot renew --dry-run

Проверьте, что в /etc/cron.d/certbot есть запись для автоматического обновления. Если нет — добавьте:

echo "0 */12 * * * root certbot renew --quiet --no-self-upgrade" | sudo tee /etc/cron.d/certbot

Раздел 10. Дополнительные виртуальные хосты

10.1. Панель x-ui (порт 4443)

Если вы планируете использовать x-ui для управления VPN, добавьте отдельный серверный блок:

server {
    listen 4443 ssl;
    server_name consol.psj-notes.ru;
    ssl_certificate /etc/letsencrypt/live/psj-notes.ru/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/psj-notes.ru/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:37254;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
