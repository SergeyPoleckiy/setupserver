x-ui + Xray: VPN-туннель с VLESS + Reality

«Иные двери открываются не ключом, а умением притвориться кем-то другим. — Воланд»

Введение

Настоящее руководство описывает установку и настройку x-ui — веб-панели для управления Xray — современным прокси, реализующим протокол VLESS с шифрованием Reality.

Мы начинаем с сервера, на котором уже работают Nginx, PHP, Grav (шаги 08–09). Сайт доступен по HTTPS на порту 8443. Порт 443 пока свободен — если не считать балансировщика Timeweb.

Цель данного руководства — установить x-ui, настроить входящее подключение VLESS + Reality на порту 443, обеспечить маскировку трафика под обычный HTTPS и создать конфигурацию для клиентов.

Входные данные: сервер доступен по SSH, UFW настроен, домен psj-notes.ru делегирован на сервер.

Раздел 1. Что такое Xray с Reality

1.1. Краткая теория

Xray — это развитие проекта V2Ray, прокси-инструмент, поддерживающий множество протоколов. VLESS — легковесный протокол без шифрования на уровне приложения (шифрование возложено на транспорт). Reality — технология маскировки трафика под TLS-соединение к реальному сайту.

Как это работает:

Клиент → TLS handshake → Сервер (порт 443)
                             ↓ Reality проверяет: настоящий TLS или VLESS?
                             ├─ Настоящий (например, браузер к grav) → Xray не отвечает
                             │  (но на 443 у нас xray, а nginx на 8443 — это нюанс)
                             └─ VLESS с Reality → Прокси-соединение

Внешний наблюдатель видит обычный TLS-трафик к реальному сайту (например, yahoo.com). Определить, что это VPN, по анализу трафика — невозможно.

1.2. Почему Reality, а не старый TLS

Подход Проблема
VMess/TLS TLS-сертификат на сервере — его можно проверить. Известные CDN-диапазоны IP.
VLESS + Reality Нет сертификата на сервере. Маскировка под чужой сайт. Обнаружить невозможно.

Раздел 2. Установка x-ui

2.1. Скрипт установки

Войдите под root или используйте sudo:

bash <(curl -Ls https://raw.githubusercontent.com/FranzKafkaYu/x-ui/refs/heads/master/install.sh)

Скрипт задаст несколько вопросов:

  • Логин и пароль для панели управления — запомните их
  • Порт панели — укажите 4443 (этот порт мы открыли в UFW)
  • Путь панели — оставьте / или укажите свой

После установки x-ui запустится и будет доступен по адресу http://<IP>:4443 или через nginx reverse proxy на https://consol.psj-notes.ru:4443.

2.2. Проверка установки

systemctl status x-ui

Должен быть active (running).

Раздел 3. Настройка через веб-панель

3.1. Вход в панель

Откройте браузер и перейдите на https://consol.psj-notes.ru:4443. Введите логин и пароль, указанные при установке.

3.2. Создание входящего подключения

В панели перейдите в раздел «Входящие подключения» → «Добавить»:

Параметр Значение Пояснение
Протокол VLESS Базовый протокол
Порт 443 Основной HTTPS-порт
Транспорт tcp Поверх TCP
Маскировка none Маскировку обеспечивает Reality
Шифрование reality Технология маскировки
Режим balance Рекомендуемый
dest yahoo.com:443 Реальный сайт для маскировки
serverNames yahoo.com, www.yahoo.com SNI-имена для маскировки
privateKey сгенерировать Ключ для Reality
shortIds оставить пустым Идентификатор клиента
Количество клиентов 1 На каждого пользователя — свой ключ

3.3. Генерация ключей

В панели есть кнопка «Сгенерировать» рядом с полем privateKey. Нажмите её — будут созданы публичный и приватный ключи Reality.

  • Приватный ключ остаётся на сервере (в конфиге x-ui)
  • Публичный ключ (id) нужно передать клиенту

3.4. Параметры клиента

В разделе клиентов заполните:

Параметр Описание
Имя Например, phone-samsung
Email Для идентификации (не обязателен для отправки)
Ключ (id) Скопируйте сгенерированный UUID клиента

После настройки нажмите «Применить конфиг».

Раздел 4. Конфигурация клиента

4.1. PC-клиент (v2rayN, Nekoray, V2Box)

Параметры подключения:

Порт 443
Протокол VLESS

4.2. iOS-клиент (Shadowrocket, Streisand, FoXray)

Настройки аналогичны. Большинство клиентов поддерживают импорт по ссылке — скопируйте ссылку из панели (кнопка «Копировать ссылку»).

4.3. Экспорт из панели

В панели x-ui нажмите на иконку QR-кода рядом с клиентом — откроется окно с конфигурацией в JSON-формате и ссылкой для импорта.

Раздел 5. Особенности работы через балансировщик Timeweb

Timeweb балансировщик перехватывает порты 80 и 443. На порту 443 он принимает TLS-соединения и проксирует их на сервер. Reality-трафик выглядит как TLS, поэтому балансировщик пропускает его.

Однако балансировщик может вмешиваться в SNI. Если вы используете домен psj-notes.ru — балансировщик направит трафик на порт 443 вашего сервера (где xray). Если вы используете photos.psj-notes.ru — трафик пойдёт на порт 80 (sslh → nginx).

Для стабильной работы VPN:

  • Используйте основной домен (psj-notes.ru) для подключения
  • Укажите в клиенте serverName = yahoo.com (не ваш домен)
  • В панели Timeweb для порта 443 должен быть указан IP сервера без дополнительной обработки

Раздел 6. Обновление x-ui

bash <(curl -Ls https://raw.githubusercontent.com/FranzKafkaYu/x-ui/refs/heads/master/install.sh)

Скрипт обнаружит существующую установку и предложит обновить её.
