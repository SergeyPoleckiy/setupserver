Grav CMS: установка и настройка блога

«Содержимое важнее формы, но без формы содержимое — всего лишь чернила на бумаге. — Воланд»

Введение

Настоящее руководство описывает установку и настройку Grav CMS — файловой системы управления контентом (flat-file CMS) для ведения блога.

Мы начинаем с сервера, на котором уже настроены Nginx, PHP-FPM и SSL-сертификаты (шаг 08). Веб-сервер готов принимать запросы, осталось установить саму CMS и наполнить её контентом.

Цель данного руководства — установить Grav, настроить его для работы на домене psj-notes.ru, подключить необходимые плагины и сделать блог доступным через панель администратора.

Входные данные: Nginx обслуживает HTTPS на порту 8443, PHP-FPM работает, сайт доступен по адресу https://psj-notes.ru:8443.

Раздел 1. Что такое Grav CMS

1.1. Краткая теория

Grav — это flat-file CMS, то есть система управления контентом, которая не использует базу данных. Все страницы хранятся в виде Markdown-файлов с YAML-заголовками в файловой системе. Это даёт ряд преимуществ:

Преимущество Пояснение
Нет БД Не нужен PostgreSQL или MySQL. Всё — файлы.
Простота бэкапа Копируете директорию — это и есть бэкап.
Скорость Нет запросов к БД. Кэширование — на уровне файлов.
Версионирование Файлы можно хранить в Git.
Git-синхронизация (опционально) Можно пушить контент напрямую в репозиторий.

1.2. Структура Grav

/var/www/psj-notes.ru/
├── user/
│   ├── pages/          # Страницы сайта (Markdown + YAML)
│   ├── config/         # Конфигурация системы, плагинов
│   ├── plugins/        # Установленные плагины
│   ├── themes/         # Установленные темы
│   └── accounts/       # Учётные записи администраторов
├── cache/              # Кэш (очищается на rebuild)
├── images/             # Кэш изображений
└── backup/             # Бэкапы (если настроены)

Раздел 2. Установка Grav

2.1. Скачивание

Перейдите в директорию веб-сервера и скачайте Grav:

cd /var/www
sudo rm -rf psj-notes.ru  # Удаляем пустую директорию, если есть
sudo wget https://getgrav.org/download/core/grav/latest -O grav.zip
sudo unzip grav.zip
sudo mv grav psj-notes.ru
sudo rm grav.zip

2.2. Права доступа

Grav должен иметь возможность писать в кэш и создавать файлы. Настроим правильные права:

sudo chown -R www-data:www-data /var/www/psj-notes.ru/
sudo chmod -R 755 /var/www/psj-notes.ru/
sudo chmod -R 775 /var/www/psj-notes.ru/cache
sudo chmod -R 775 /var/www/psj-notes.ru/images
sudo chmod -R 775 /var/www/psj-notes.ru/backup
sudo chmod -R 775 /var/www/psj-notes.ru/user/data

2.3. Проверка

Откройте браузер и перейдите на https://psj-notes.ru:8443. Вы должны увидеть приветственную страницу Grav. Если видите её — установка прошла успешно.

Раздел 3. Настройка Grav

3.1. Базовая конфигурация

Отредактируйте /var/www/psj-notes.ru/user/config/system.yaml:

absolute_urls: false
timezone: Europe/Moscow
default_locale: ru
param_sep: ':'
reverse_proxy_setup: false
force_ssl: true
force_lowercase_urls: true
custom_base_url: 'https://psj-notes.ru'
username_regex: '^[a-z0-9_-]{3,16}$'
pwd_regex: '(?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{8,}'
intl_enabled: true
http_x_forwarded:
  protocol: true
  host: false
  port: true
  ip: true
languages:
  supported:
    - ru

3.2. Ключевые параметры

Параметр Значение Зачем
timezone: Europe/Moscow Московское время Даты публикаций
custom_base_url https://psj-notes.ru Абсолютные ссылки в Grav
force_ssl: true Включено Принудительный HTTPS
languages.supported: [ru] Русский язык Одной языковой версии достаточно

Раздел 4. Плагины

Для полноценной работы Grav рекомендуется установить минимальный набор плагинов. Наиболее важные — Admin Panel (веб-интерфейс управления) и Email (отправка писем через Postfix).

4.1. Установка плагинов через GPM

Grav Package Manager (GPM) — встроенная утилита для управления плагинами:

cd /var/www/psj-notes.ru

# Admin Panel — веб-интерфейс
sudo -u www-data bin/gpm install admin

# Email — для отправки уведомлений (сброс пароля и т.д.)
sudo -u www-data bin/gpm install email

# Login — страница авторизации
sudo -u www-data bin/gpm install login

# Form — для создания форм
sudo -u www-data bin/gpm install form

4.2. Настройка Email

После установки плагина Email настройте его в файле /var/www/psj-notes.ru/user/config/plugins/email.yaml:

enabled: true
mailer:
  engine: smtp
  smtp:
    server: localhost
    port: 25
    encryption: none
  sendmail:
    bin: '/usr/sbin/sendmail -bs'

Или, если Postfix настроен на отправку через Яндекс (шаг 07), можно указать relay напрямую:

enabled: true
mailer:
  engine: smtp
  smtp:
    server: smtp.yandex.ru
    port: 587
    encryption: tls
    user: 'sergey@poleckiy.ru'
    password: 'пароль_приложения'

4.3. Список установленных плагинов на этом сервере

admin, email, error, flex-objects, form, login, markdown-notices, problems

Раздел 5. Создание администратора

5.1. Через командную строку

Перейдите в директорию Grav и выполните:

cd /var/www/psj-notes.ru
sudo -u www-data bin/plugin login newuser

Следуйте инструкциям: укажите username, полное имя, пароль и email. После создания вы сможете войти в админ-панель по адресу https://psj-notes.ru:8443/admin.

5.2. Через веб-интерфейс

Если вы ещё не создали администратора, Grav предложит создать его при первом посещении /admin. Просто заполните форму.

Раздел 6. Создание страниц

6.1. Структура страниц

Grav хранит страницы в /var/www/psj-notes.ru/user/pages/. Каждая страница — это директория с файлом default.md.

Пример структуры:

user/pages/
├── 01.home/                    # Главная страница (порядок: 01)
│   └── default.md
└── 02.configuring_the_vds_server_as_a_vpn/  # Раздел (порядок: 02)
    ├── default.ru.md           # Введение в раздел
    ├── 02.ot-vds-do-ssh/       # Шаг 01
    │   └── default.ru.md
    ├── 03.nastroika-ssh-zsh/   # Шаг 02
    │   └── default.ru.md
    └── ...

Нумерация папок (01., 02....) определяет порядок сортировки в меню.

6.2. Формат Markdown

Каждый файл начинается с YAML-заголовка (frontmatter) и содержит тело в Markdown:

---
title: 'Заголовок страницы'
---
# Основной заголовок

Текст страницы в Markdown.

Раздел 7. Кэш

После любых изменений конфигурации или контента очищайте кэш Grav:

cd /var/www/psj-notes.ru
sudo -u www-data bin/grav clear-cache

Это сбросит кэш и применит все изменения.

Раздел 8. Обновление Grav

Регулярно проверяйте доступные обновления:

cd /var/www/psj-notes.ru
sudo -u www-data bin/gpm update

Для обновления самого Grav:

sudo -u www-data bin/gpm selfupgrade
