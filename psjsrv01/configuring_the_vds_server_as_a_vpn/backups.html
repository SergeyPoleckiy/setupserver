Бэкапы: автоматическое резервное копирование

«Рукописи не горят. А вот серверы — ещё как. — Воланд, перебирая страницы»

Введение

Настоящее руководство описывает настройку автоматического резервного копирования критически важных данных сервера: файлов Grav CMS, конфигурации x-ui и системных конфигурационных файлов.

Мы начинаем с сервера, на котором уже работают все сервисы (шаги 01–10). Сервер принимает сайт, раздаёт VPN, отправляет почту. Пришло время обеспечить сохранность данных.

Цель данного руководства — настроить ежедневные и еженедельные бэкапы, отправлять отчёты на почту и хранить ротацию копий.

Входные данные: Postfix настроен и отправляет почту (шаг 07), Grav CMS установлен (шаг 09), x-ui работает (шаг 10).

Раздел 1. Что и как бэкапим

1.1. Стратегия

Данные Периодичность Хранение Размер
Grav CMS (файлы, контент, плагины) Ежедневно 7 копий ~50–200 MB
Конфигурация x-ui (база данных) Ежедневно 14 копий ~1–5 MB
Системные конфиги (nginx, postfix, ufw, wireguard) Еженедельно 4 копии ~1 MB

1.2. Куда сохраняем

Бэкапы хранятся локально в /var/backups/. Для критически важных данных рекомендуется также настроить копирование на внешнее хранилище (например, S3, rsync на другой сервер).

Раздел 2. Скрипты бэкапов

2.1. Бэкап Grav CMS

Создайте /usr/local/bin/backup-grav.sh:

#!/bin/bash
BACKUP_DIR="/var/backups/grav"
TIMESTAMP=$(date +%Y%m%d-%H%M%S)
KEEP=7

mkdir -p "$BACKUP_DIR"

tar czf "$BACKUP_DIR/grav-$TIMESTAMP.tar.gz" \
    -C /var/www/psj-notes.ru \
    --exclude=cache \
    --exclude=images \
    --exclude=tmp \
    user/ config/ composer.json composer.lock

# Удалить старые бэкапы (оставить только KEEP последних)
ls -t "$BACKUP_DIR"/grav-*.tar.gz 2>/dev/null | tail -n +$((KEEP+1)) | xargs -r rm

echo "Grav backup: grav-$TIMESTAMP.tar.gz ($(du -h "$BACKUP_DIR/grav-$TIMESTAMP.tar.gz" | cut -f1))"

2.2. Бэкап x-ui

Создайте /usr/local/bin/backup-xui.sh:

#!/bin/bash
BACKUP_DIR="/var/backups/xui"
TIMESTAMP=$(date +%Y%m%d-%H%M%S)
KEEP=14

mkdir -p "$BACKUP_DIR"

# База данных x-ui (SQLite)
if [ -f /etc/x-ui/x-ui.db ]; then
    cp /etc/x-ui/x-ui.db "$BACKUP_DIR/xui-$TIMESTAMP.db"
elif [ -f /usr/local/x-ui/bin/x-ui.db ]; then
    cp /usr/local/x-ui/bin/x-ui.db "$BACKUP_DIR/xui-$TIMESTAMP.db"
fi

# Конфигурация xray
if [ -f /usr/local/x-ui/bin/config.json ]; then
    cp /usr/local/x-ui/bin/config.json "$BACKUP_DIR/xray-config-$TIMESTAMP.json"
fi

# Удалить старые бэкапы
ls -t "$BACKUP_DIR"/xui-*.db 2>/dev/null | tail -n +$((KEEP+1)) | xargs -r rm
ls -t "$BACKUP_DIR"/xray-config-*.json 2>/dev/null | tail -n +$((KEEP+1)) | xargs -r rm

echo "x-ui backup: xui-$TIMESTAMP.db ($(du -h "$BACKUP_DIR/xui-$TIMESTAMP.db" | cut -f1))"

2.3. Бэкап системных конфигов

Создайте /usr/local/bin/backup-system.sh:

#!/bin/bash
BACKUP_DIR="/var/backups/system"
TIMESTAMP=$(date +%Y%m%d-%H%M%S)
KEEP=4

mkdir -p "$BACKUP_DIR"

tar czf "$BACKUP_DIR/system-$TIMESTAMP.tar.gz" \
    /etc/nginx/ \
    /etc/postfix/ \
    /etc/ufw/ \
    /etc/ssh/sshd_config \
    /etc/fail2ban/ \
    /etc/default/sslh \
    /etc/wireguard/ \
    /etc/apt/apt.conf.d/20auto-upgrades \
    /etc/apt/apt.conf.d/50unattended-upgrades

ls -t "$BACKUP_DIR"/system-*.tar.gz 2>/dev/null | tail -n +$((KEEP+1)) | xargs -r rm

echo "System backup: system-$TIMESTAMP.tar.gz ($(du -h "$BACKUP_DIR/system-$TIMESTAMP.tar.gz" | cut -f1))"

2.4. Права на выполнение

sudo chmod +x /usr/local/bin/backup-{grav,xui,system}.sh

Раздел 3. Скрипт полного бэкапа

Создайте /usr/local/bin/backup-all.sh — единый скрипт, который запускает все три и отправляет отчёт:

#!/bin/bash

REPORT=$(mktemp)

echo "=== Backup report: $(date) ===" >> "$REPORT"

/usr/local/bin/backup-grav.sh >> "$REPORT" 2>&1
/usr/local/bin/backup-xui.sh >> "$REPORT" 2>&1

# Системный — только по воскресеньям
if [ "$(date +%u)" -eq 7 ]; then
    /usr/local/bin/backup-system.sh >> "$REPORT" 2>&1
fi

# Отправить отчёт на почту
cat "$REPORT" | sudo mail -s "Backup report: $(hostname) — $(date +%Y-%m-%d)" sergey@poleckiy.ru

rm "$REPORT"
sudo chmod +x /usr/local/bin/backup-all.sh

Раздел 4. Настройка cron

4.1. Ежедневный бэкап в 3:00

sudo tee /etc/cron.d/backups << 'EOF'
# Ежедневный бэкап Grav, x-ui и системы (по воскресеньям)
0 3 * * * root /usr/local/bin/backup-all.sh
EOF

4.2. Проверка

sudo systemctl restart cron
sudo run-parts --test /etc/cron.d  # Проверить синтаксис

Раздел 5. Восстановление из бэкапа

5.1. Grav

sudo tar xzf /var/backups/grav/grav-20260708-030001.tar.gz -C /var/www/psj-notes.ru/
sudo chown -R www-data:www-data /var/www/psj-notes.ru/

5.2. x-ui

sudo systemctl stop x-ui
sudo cp /var/backups/xui/xui-20260708-030001.db /usr/local/x-ui/bin/x-ui.db
sudo systemctl start x-ui

5.3. Системные конфиги

sudo tar xzf /var/backups/system/system-20260708-030001.tar.gz -C /

Будьте осторожны — tar восстановит файлы поверх существующих. Лучше распаковать в отдельную директорию и копировать вручную.

Раздел 6. Мониторинг

Проверьте, что письма-отчёты приходят на почту. Если письмо не пришло — проверьте Postfix:

sudo tail -20 /var/log/mail.log

И запустите бэкап вручную для диагностики:

sudo /usr/local/bin/backup-all.sh

«Теперь ваши данные в безопасности, — подытожил Воланд. — По крайней мере, до тех пор, пока вы помните, где лежит ключ от сейфа. А ключ, как известно, должен быть не в сейфе.»
