# Базовая система: локаль, время, имя, память

> *«Система без базовой настройки — это как человек без паспорта и часов. Вроде живёт, а кто он и когда — неизвестно.» — Воланд*

## Введение

Настоящее руководство настраивает фундамент сервера, на который лягут все остальные сервисы. Мы начинаем с уже работающего SSH-сервера, к которому можем подключиться пользователем `user01` и имеем права `sudo` без пароля.

По сравнению с предыдущим документом сервер всё ещё «голый»: не установлена правильная локаль, часовой пояс, скорее всего, стандартный европейский или UTC, hostname — набор символов, выданных провайдером, а swap-файл отсутствует. На сервере с **2 ГБ оперативной памяти** отсутствие swap — это как ходить по канату без страховки. Любой всплеск трафика или запуск тяжёлого скрипта приведёт к падению важнейших служб.

Все шаги пронумерованы и должны выполняться **в указанном порядке**.

> **Входные данные:** вы подключены к серверу `srv02` по SSH пользователем `user01`. Команды с `sudo` не запрашивают пароль.

## Раздел 1. Настройка локали

### 1.1. Что такое локаль

**Локаль (locale)** — набор параметров, определяющих язык, кодировку, форматирование дат, чисел и валют для программ в системе. Без правильно настроенной локали вы получите «кракозябры» в логах, некорректное отображение кириллицы и проблемы с сортировкой текста.

В Ubuntu по умолчанию часто установлена только английская локаль `en_US.UTF-8`. Мы добавим `ru_RU.UTF-8` и сделаем её основной.

### 1.2. Проверка состояния

Проверьте текущие настройки:

```bash
locale
```

Если вы видите значения `C` или `en_US.UTF-8` — систему нужно русифицировать. Проверьте список уже сгенерированных локалей:

```bash
locale -a
```

### 1.3. Генерация и применение `ru_RU.UTF-8`

Установите пакет локалей и сгенерируйте русскую локаль:

```bash
sudo apt install -y locales
sudo locale-gen ru_RU.UTF-8
```

Настройте её как системную локаль по умолчанию:

```bash
sudo update-locale LANG=ru_RU.UTF-8 LANGUAGE=ru_RU:ru LC_ALL=ru_RU.UTF-8
```

Примените изменения в текущей сессии:

```bash
export LANG=ru_RU.UTF-8
export LANGUAGE=ru_RU:ru
export LC_ALL=ru_RU.UTF-8
```

Убедитесь, что все переменные локали теперь указывают на `ru_RU.UTF-8` с помощью команды `locale`.

> *Воланд удовлетворённо улыбнулся:*
> *«Теперь сервер понимает, на каком языке с ним говорят. Раньше он был как иностранец, не знающий ни одного наречия. Теперь — он наш. Русский, грамотный, с правильной кодировкой. И дата будет отображаться так, как привыкли мы, а не какой-нибудь техасец.»*

## Раздел 2. Часовой пояс и время

### 2.1. Часовой пояс и NTP

**Часовой пояс (timezone)** определяет местное время на сервере. По умолчанию сервера часто работают в UTC. Это неудобно: логи, планировщик cron, время создания файлов — всё будет показывать чужое время.

**NTP (Network Time Protocol)** — протокол синхронизации системных часов с серверами точного времени. В Ubuntu 24.04 по умолчанию запущена служба `systemd-timesyncd`, которая отлично справляется с задачей. Настраивать NTP вручную не требуется. Убедитесь в синхронизации:

```bash
timedatectl show --property=NTPSynchronized --value
```

Если вернулось `yes` — часы синхронизируются автоматически.

### 2.2. Установка часового пояса

Проверьте текущее время и зону:

```bash
timedatectl
```

Установите часовой пояс (например, московское время):

```bash
sudo timedatectl set-timezone Europe/Moscow
```

Проверьте результат с помощью `timedatectl`. Время `Local time` должно отображать текущее московское время (MSK, +0300).

> *Коровьев прищурился на часы:*
> *«Время — вещь удивительная. В Амстердаме полдень, а у нас уже три часа дня. Но серверу, видите ли, всё равно — он работает и в UTC, и в MSK. Мы ставим время не ради сервера, а для нашего удобства: чтобы логи показывали, когда именно Анна зашла на сервер, а не когда солнце в Лондоне было в зените.»*

## Раздел 3. Установка Hostname

### 3.1. Зачем менять имя

По умолчанию провайдеры дают серверам безликие имена типа `vps-12345`. Мы назовём сервер осмысленно — **`srv02`**, чтобы при входе в консоль сразу понимать, на каком узле мы находимся.

### 3.2. Изменение имени хоста

Задайте статическое имя сервера:

```bash
sudo hostnamectl set-hostname srv02
```

### 3.3. Обновление `/etc/hosts`

После изменения hostname необходимо обновить локальную базу имён. Откройте файл:

```bash
cat /etc/hosts
```

Найдите строчку, сопоставленную с `127.0.1.1` (там может быть старое имя типа `vps-xxxxx`), и замените его на `srv02`:

```bash
sudo sed -i 's/127\.0\.1\.1.*/127.0.1.1\tsrv02/' /etc/hosts
```

Убедитесь, что имя успешно применилось командой `hostnamectl`.

> *Воланд задумчиво кивнул:*
> *«Теперь у сервера есть имя. Не какое-то "vps-12345", а настоящее — srv02. Знаете, это важно — знать, как зовут того, кому вы поручаете свои тайны. Бездомные, кстати, тоже сначала дают кличку, а потом уже доверяют.»*

## Раздел 4. Настройка swap-файла (память подкачки)

### 4.1. Зачем нужен swap на 2 ГБ RAM

**Swap (файл подкачки)** — область на диске, используемая ядром в качестве расширения оперативной памяти. Наш сервер располагает **2 ГБ оперативной памяти**. Для базовых утилит этого хватает, но запуск VPN-служб, CMS и веб-сервера может быстро исчерпать этот лимит.

Без swap-файла при заполнении RAM сработает механизм ядра **OOM-killer** (Out Of Memory killer). Он просто завершит самый крупный процесс, чтобы спасти операционную систему от зависания. OOM-killer с удовольствием «пристрелит» базу данных или VPN-сервис.

Мы создадим swap-файл размером **4 ГБ** (оптимальный размер для системы с 2 ГБ RAM).

### 4.2. Создание файла подкачки

Выделите место под файл подкачки размером 4 ГБ:

```bash
sudo fallocate -l 4G /swapfile
```

Проверьте размер созданного файла:

```bash
ls -lh /swapfile
```

### 4.3. Ограничение прав доступа

Swap-файл содержит дампы памяти, в которых могут находиться чувствительные данные (пароли, ключи шифрования). Файл должен быть доступен только суперпользователю `root`:

```bash
sudo chmod 600 /swapfile
```

### 4.4. Активация swap-пространства

Отформатируйте файл под нужды подкачки:

```bash
sudo mkswap /swapfile
```

Включите swap-пространство в системе:

```bash
sudo swapon /swapfile
```

Убедитесь, что система увидела файл подкачки:

```bash
sudo swapon --show
free -h
```

### 4.5. Автоматическое монтирование при старте

Чтобы swap не отключился после перезагрузки сервера, добавьте запись в файл `/etc/fstab`:

```bash
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

### 4.6. Тонкая настройка swappiness

Параметр ядра **`vm.swappiness`** (значения от 0 до 100) определяет, насколько охотно система переносит данные из RAM в swap. Значение по умолчанию в Ubuntu — `60` (слишком агрессивный перенос на диск). 

Для сервера с 2 ГБ оперативной памяти оптимально выставить значение `10` — это заставит ядро использовать быструю RAM на максимум, задействуя медленный диск только при реальной нехватке памяти.

Установите значение:

```bash
sudo sysctl vm.swappiness=10
```

Для сохранения настройки после перезагрузки создайте конфигурационный файл:

```bash
echo 'vm.swappiness=10' | sudo tee /etc/sysctl.d/99-swappiness.conf
```

> *Воланд предупреждает:*
> *«Swap — это не замена оперативной памяти. Это последняя линия обороны. Если ваш сервер постоянно работает с заполненным swap — это как дышать через трубочку: жив, но качество жизни… нет, вам нужен сервер с большей RAM.»*

## Раздел 5. Обновление пакетов операционной системы

### 5.1. Зачем обновляться

Предоставленный провайдером образ ОС может содержать устаревшие пакеты с известными уязвимостями. Перед развёртыванием целевых служб обновите систему.

### 5.2. Обновление и очистка

Выполните обновление списков пакетов и установку новых версий программ:

```bash
sudo apt update
sudo apt upgrade -y
```

После завершения удалите ненужные остатки пакетов для экономии места на диске:

```bash
sudo apt autoremove -y && sudo apt autoclean
```

> *Воланд, просматривая список обновлённых пакетов:*
> *«Сорок сороков обновлений. Сорок сороков дыр, которые кто-то уже закрыл, пока вы читали это руководство. Сервер — как квартира: чем дольше не закрываете замки, тем больше желающих зайти без приглашения.»*
