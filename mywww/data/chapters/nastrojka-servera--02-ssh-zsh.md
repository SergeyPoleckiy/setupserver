# Настройка SSH, zsh и удобства работы на сервере

> *«Комфорт — вещь субъективная, — заметил Воланд, поправляя воротник. — Но ключи в двери — это объективно.»*

## Введение

Настоящее руководство продолжает настройку сервера, начатую в предыдущем документе. Мы начинаем с уже работающего SSH-сервера, к которому можем подключиться по паролю под пользователем `user01`.

Цель данного руководства — за один последовательный проход выполнить три группы задач:

1.  **Временно разрешить `sudo` без пароля** — для удобства последующей настройки (в будущем это будет пересмотрено).
2.  **Настроить SSH-ключи** — безопасный переход от паролей к криптографическим ключам.
3.  **Установить и настроить zsh** — оболочку с темой powerlevel10k, шрифтами и плагинами автодополнения для комфортной работы.

Все шаги пронумерованы и должны выполняться **в указанном порядке**.

> **Входные данные:** вы подключены к серверу `srv02` по SSH пользователем `user01` с использованием пароля.

## Раздел 1. Временная настройка `sudo` без пароля

### 1.1. Зачем это нужно

В процессе развёртывания систем мы выполним десятки команд через `sudo`. Постоянный ввод пароля замедляет работу и утомляет. Мы временно отключим запрос пароля для `user01`.

### 1.2. Создание файла `sudoers`

Откройте сессию суперпользователя:

```bash
sudo -i
```

Система запросит ваш пароль. Введите его. После переключения в режим `root` (приглашение `#`) создайте конфигурационный файл:

```bash
echo 'user01 ALL=(ALL) NOPASSWD:ALL' > /etc/sudoers.d/user01
chmod 440 /etc/sudoers.d/user01
```

Проверьте синтаксис файлов конфигурации `sudo`:

```bash
visudo -c
```

Ожидаемый вывод:
```
/etc/sudoers: parsed OK
/etc/sudoers.d/user01: parsed OK
```

Если ошибок не обнаружено, выйдите из сессии `root`:

```bash
exit
```

### 1.3. Проверка

Убедитесь, что команды с `sudo` выполняются мгновенно без запроса пароля:

```bash
sudo whoami
```

Должно вернуться:
```
root
```

> *«Открытая дверь — удобная вещь для хозяина. И столь же удобная для всех гостей, включая непрошеных. Пользуйтесь временно, пока настраиваете хозяйство. А потом — закройте, пожалуйста.»*

## Раздел 2. Генерация SSH-ключа на рабочей станции

### 2.1. Анатомия ключей

Пара ключей состоит из двух частей:
-   **Приватный ключ** (`id_ed25519`) — хранится **строго на вашем компьютере** в секрете.
-   **Публичный ключ** (`id_ed25519.pub`) — отправляется на сервер. По нему сервер проверяет вашу подлинность.

Алгоритм **Ed25519** является современным стандартом — он быстрее и безопаснее старого RSA.

### 2.2. Генерация

Этот шаг выполняется **на вашем персональном компьютере** (локальном терминале), а не на сервере!

```bash
ssh-keygen -t ed25519 -C "user01@srv02"
```

При запросе пути сохранения нажмите Enter (по умолчанию используется `~/.ssh/id_ed25519`).  
При запросе кодовой фразы (passphrase) вы можете нажать Enter дважды для беспарольного входа по ключу или задать пароль для дополнительной защиты самого файла ключа.

## Раздел 3. Копирование публичного ключа на сервер

### 3.1. Использование `ssh-copy-id`

На вашем локальном компьютере выполните:

```bash
ssh-copy-id user01@<IP-адрес-сервера>
```

Введите пароль пользователя `user01`. Утилита автоматически добавит ваш публичный ключ в файл `~/.ssh/authorized_keys` на сервере.

### 3.2. Проверка

Попробуйте войти на сервер заново:

```bash
ssh user01@<IP-адрес-сервера>
```

Если всё сделано правильно, сервер впустит вас без запроса пароля.

> *Коровьев, подмигивая:*
> *«Ключик-то у вас — золотой. Только дверь ещё не заменили. Сейчас поменяем — и старый пароль станет таким же бесполезным, как прошлогодний снег.»*

## Раздел 4. Отключение входа по паролю на сервере

### 4.1. Изменение `sshd_config`

Чтобы полностью защититься от брутфорса, отключим аутентификацию по паролям.

На сервере сделайте резервную копию конфигурационного файла:

```bash
sudo cp /etc/ssh/sshd_config /etc/ssh/sshd_config.backup.$(date +%Y%m%d_%H%M%S)
```

Откройте файл для редактирования:

```bash
sudo nano /etc/ssh/sshd_config
```

Найдите и установите следующие параметры (раскомментируйте их при необходимости):

```
PasswordAuthentication no
PubkeyAuthentication yes
KbdInteractiveAuthentication no
UsePAM yes
MaxAuthTries 3
```

Сохраните файл (`Ctrl+O`, Enter) и закройте редактор (`Ctrl+X`).

### 4.2. Настройка прав доступа

SSH-сервер заблокирует вход по ключу, если права на директории настроены некорректно. Обеспечьте безопасность:

```bash
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
chmod 755 ~
```

### 4.3. Применение изменений

Проверьте конфигурационный файл на синтаксические ошибки:

```bash
sudo sshd -t
```

Если вывод пуст, примените настройки. В Ubuntu 24.04 служба SSH управляется сокетами, поэтому выполните перезапуск следующим образом:

```bash
sudo systemctl daemon-reload
sudo systemctl restart ssh.socket
sudo systemctl restart ssh.service
```

### 4.4. Тестирование соединения

**Важно: Не закрывайте текущую сессию терминала!** Откройте новое окно на своём компьютере и попробуйте подключиться:

```bash
ssh user01@<IP-адрес-сервера>
```

Если подключение прошло успешно, значит, всё настроено верно. Старую сессию теперь можно закрыть.

## Раздел 5. Установка и настройка zsh

### 5.1. Зачем менять оболочку

Стандартный Bash надёжен, но **zsh** предлагает гораздо больше удобств: интеллектуальное автодополнение, исправление опечаток «на лету» и великолепные темы оформления.

### 5.2. Установка зависимостей

```bash
sudo apt install -y zsh git curl unzip fontconfig
```

### 5.3. Установка oh-my-zsh

Фреймворк **oh-my-zsh** упрощает управление плагинами и темами:

```bash
sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)" "" --unattended
```

Установите zsh оболочкой по умолчанию для `user01`:

```bash
sudo chsh -s $(which zsh) user01
```

### 5.4. Установка темы powerlevel10k

Установите популярную тему оформления:

```bash
git clone --depth=1 https://github.com/romkatv/powerlevel10k.git ${ZSH_CUSTOM:-$HOME/.oh-my-zsh/custom}/themes/powerlevel10k
```

Откройте ваш `.zshrc`:

```bash
nano ~/.zshrc
```

Измените значение переменной темы на:

```
ZSH_THEME="powerlevel10k/powerlevel10k"
```

### 5.5. Шрифты Nerd Fonts

Тема powerlevel10k использует красивые иконки. Чтобы они отображались корректно:
1. Скачайте шрифт **MesloLGS NF** на свой локальный компьютер (ссылка: [MesloLGS NF.zip](https://github.com/romkatv/powerlevel10k-media/raw/master/MesloLGS%20NF.zip)).
2. Распакуйте и установите шрифт в вашей операционной системе.
3. В настройках вашего терминала (iTerm2, Windows Terminal, Kitty и т.д.) выберите шрифт **MesloLGS NF** в качестве основного.

### 5.6. Установка плагинов

Добавим полезные плагины для автоматического дополнения и подсветки синтаксиса:

```bash
# Автодополнение из истории команд
git clone https://github.com/zsh-users/zsh-autosuggestions.git ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-autosuggestions

# Подсветка синтаксиса (зелёный — команда верна, красный — ошибочна)
git clone https://github.com/zsh-users/zsh-syntax-highlighting.git ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-syntax-highlighting

# Всплывающее меню вариантов
git clone --depth 1 https://github.com/marlonrichert/zsh-autocomplete.git ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-autocomplete
```

Активируйте их в файле `~/.zshrc`, заменив строчку `plugins=(git)` на следующую:

```
plugins=(git zsh-autosuggestions zsh-syntax-highlighting zsh-autocomplete)
```

Сохраните изменения и выйдите.

### 5.7. Первый запуск и конфигурация

Запустите zsh:

```bash
zsh
```

Тема `powerlevel10k` сразу запустит интерактивный мастер настройки. Ответьте на несколько вопросов, чтобы настроить стиль приветствия по своему вкусу.

## Раздел 6. Настройка приветственного баннера и SSH-алиаса

### 6.1. Замена приветствия сервера (MOTD)

При входе на сервер система показывает вам так называемый MOTD (Message of the Day) — сообщение дня. По умолчанию Ubuntu выводит ворох стандартных уведомлений о версии ОС, доступных обновлениях и подсказках.

Наш сервер достоин лучшего приёма.

Заменим казёнщину на приветствие в стиле нашего литературного вечера. Создайте скрипт приветствия на сервере:

```bash
sudo tee /etc/update-motd.d/00-voland-banner > /dev/null << 'BANNER_EOF'
#!/bin/bash
cat << "EOF"
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│   Добро пожаловать, дорогой гость.                               │
│                                                                  │
│   Квартирный вопрос, как известно, испортил не только москвичей, │
│   но и многих, кто пытался зайти на этот сервер без приглашения. │
│                                                                  │
│   У вас есть ключ? Отлично. Вы — свой.                           │
│   А если нет — Аннушка масло уже разлила,                        │
│   и вам лучше не скользить.                                      │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
EOF
BANNER_EOF
sudo chmod +x /etc/update-motd.d/00-voland-banner
```

После этого отключите все стандартные скрипты приветствия, чтобы наш баннер не тонул в шуме:

```bash
cd /etc/update-motd.d && sudo chmod -x 00-header 10-help-text 50-landscape-sysinfo 50-motd-news 85-fwupd 90-updates-available 91-contract-ua-esm-status 91-release-upgrade 92-unattended-upgrades 95-hwe-eol 97-overlayroot 98-fsck-at-reboot 98-reboot-required
```

Проверьте результат:

```bash
run-parts /etc/update-motd.d/
```

Ожидаемый вывод — одиноко парящий баннер в обрамлении тонких линий.

### 6.2. Создание короткого алиаса `ssh vpnserver`

Чтобы входить на сервер короткой командой, настройте **ваш локальный компьютер** (не сервер). Отредактируйте или создайте файл `~/.ssh/config`:

```bash
nano ~/.ssh/config
```

Добавьте в него следующий блок:

```
Host vpnserver
    HostName <IP-адрес-сервера>
    User user01
    Port 22
    IdentityFile ~/.ssh/id_ed25519
```

Сохраните файл. Теперь вы можете входить на сервер одной командой:

```bash
ssh vpnserver
```

И в ответ получить достойное приветствие.

> *Воланд улыбнулся в тени плаща:*
> *«Коротко, изящно и со смыслом. Ключ в кармане, алиас в памяти терминала, а сервер у ваших ног. Именно так и должно выглядеть знакомство с приличной машиной.»*
